System Overview

Project Summary

This document defines the complete electrical and electronics system for F.U.B.A.R., a 22-foot Calkins Bartender restoration. It serves as the master reference for system planning, design, installation, testing, documentation, operation, troubleshooting, and future maintenance.

Primary Mission

Build a dependable electronics package optimized for the Oregon coast while remaining equally capable on inland lakes such as Odell Lake.

Design Philosophy

Keep the helm clean and traditional. Retain analog engine instruments for redundancy while adding modern digital capabilities through a Garmin chartplotter and NMEA 2000 network. Freeze major design decisions once approved and build future work from that baseline.

Electrical Architecture

Separate Engine and House power buses, and charge from smart battery isolators when using shore power or the alternator while under way. Use marine-grade wiring, quality circuit protection, and reserve capacity to supply potential future radar, heading sensor,and autopilot.

Engine Monitoring Goals

Display RPM, oil pressure, coolant temperature, battery voltage, fuel level, fuel flow, fuel used, and range on the Garmin. Install independent low-oil-pressure and high-temperature alarms.

Frozen Design Decisions

  • Garmin GPSMAP 942xsv chartplotter
  • Airmar B175M 0° tilted-element through-hull transducer
  • ICOM M510BB black-box VHF with CommandMic
  • Dedicated Class B AIS transceiver
  • NMEA 2000 backbone
  • Analog tachometer, oil pressure, water temperature, voltmeter and fuel gauge retained
  • Digital engine monitoring added while preserving analog gauges

Phased Installation of Electronics

Phase 1 – Essential Vessel Systems

Objective: Make the boat electrically safe and operational.
  • Install battery system and battery switch.
  • Install DC power distribution.
  • Install navigation, anchor, courtesy, and cabin lighting.
  • Install bilge pump circuits.
  • Install USB and 12-volt accessory outlets.
Milestone: The boat is electrically safe, protected, and capable of basic operation.

Phase 2 – Helm & Navigation

Objective: Create a functional helm for safe vessel operation.
  • Install helm layout and instrument panel.
  • Install Garmin GPSMAP 942xsv chartplotter.
  • Install NMEA 2000 backbone.
  • Install engine monitoring interface and sensors.
  • Verify engine data is displayed correctly on the chartplotter.
Milestone: The vessel can be safely navigated using GPS while continuously monitoring engine health.

Phase 3 – Communications

Objective: Provide VHF communication capability.
  • Install ICOM M510BB black-box VHF radio.
  • Install ICOM HM-195GB CommandMic.
  • Verify DSC and NMEA 2000 communications.
Milestone: The vessel can communicate with other vessels and shore stations.

Phase 4 – Commissioning

Objective: Validate the complete electrical system.
  • Verify all electrical circuits.
  • Verify battery charging from the alternator and shore charger.
  • Verify NMEA 2000 network integrity.
  • Conduct dockside testing.
  • Conduct sea trials.
  • Correct deficiencies.
Milestone: The electrical and electronics systems are fully commissioned and ready for regular operation.

Phase 5 – Future Enhancements

Objective: Expand system capabilities.
  • Install Garmin AIS 800 transceiver (collision avoidance).
  • Install Garmin GMR 18 xHD3 radar.
  • Verify AIS targets and radar overlay functions on chartplotter.
Milestone: Full navigation and situational awareness suite complete.

Airmar B175M Installation

Installation Notes

  • Install in a solid laminate area near the keel line. Avoid areas with air bubbles, water flow turbulence, or directly behind hull steps.
  • Ensure the path inside the hull is clear of any stringers, bulkheads, or electrical wiring.
  • Apply a generous bead of high-quality marine sealant (e.g., 3M 4200 or 5200) to both the transducer flange and the hull opening.
  • Do not over-tighten the bronze locking nut, as this can damage the mahogany hull structure or strip the threads.
  • Torque the lock nut to 25–30 ft-lbs using a custom spanner wrench or large adjustable wrench.
  • Verify correct transducer tilt angle (e.g., 0°, 12°, or 20° version matches deadrise) before tightening.
  • Connect the Airmar B175M CHIRP Transducer directly to the Garmin GPSMAP 942xsv using the transducer cable. The transducer is not an NMEA 2000 device and does not connect to the NMEA 2000 backbone. The chartplotter processes the sonar data and can share depth information with other compatible devices over the NMEA 2000 network.

NMEA 2000 Network Data Flow

The NMEA 2000 network allows all compatible devices aboard F.U.B.A.R. to share information over a common communications backbone. Engine, fuel, navigation, and communication data are available to every connected device without requiring separate wiring between components. Each device can transmit data, receive data, or both, depending on its function.

Data Flow

  • Engine sensors send analog signals to the Engine Gateway.
  • The Engine Gateway converts the analog signals into NMEA 2000 data.
  • Fuel tank senders provide fuel level information for both tanks.
  • The NMEA 2000 backbone distributes data to all connected devices.
  • The Garmin GPSMAP 942xsv displays navigation, engine, fuel, and AIS information.
  • The Garmin AIS 800 shares vessel position and receives AIS targets from nearby vessels.
  • The ICOM M510BB VHF radio uses GPS position from the network for DSC emergency calling and position reporting.
  • A PC or laptop can connect through the NMEA 2000 USB adapter for diagnostics, configuration, software updates, and troubleshooting.

NMEA 2000 Network Data Flow

Best Practices

  • One sensor can provide data to multiple devices.
  • The chartplotter displays network data but is not required for the network to operate.
  • If one device fails or is turned off, the remaining network continues to function.
  • Verify each device is recognized on the network during installation.
  • Keep all connected devices updated with the latest firmware.
  • Document all connected devices and their network addresses for future maintenance.

Information Shared Across the Network

  • GPS Position
  • Course Over Ground (COG)
  • Speed Over Ground (SOG)
  • AIS Targets
  • Engine RPM
  • Oil Pressure
  • Coolant Temperature
  • Alternator Voltage
  • Fuel Tank Levels
  • DSC Position Data
  • Navigation Alarms and System Status

NMEA 2000 Backbone Layout

The NMEA 2000 backbone is the communication network that connects all of the electronic systems aboard F.U.B.A.R. It allows compatible devices to share navigation, engine, fuel, AIS, and communications data over a single network. The backbone is designed to be simple, reliable, and easy to expand as new equipment is added.

Best Practices

  • Use one continuous NMEA 2000 backbone—do not create branches or loops.
  • Install a 120 Ω terminator at each end of the backbone.
  • Connect each device to its own T-connector and drop cable.
  • Power the network from one fused 12-volt power tee only.
  • Mount the backbone in a dry, protected location near the helm whenever possible.
  • Keep drop cables under 20 ft (6 m).
  • Support and secure all cables to prevent vibration and chafing.
  • Keep network cables away from ignition wiring and other high-current electrical cables.
  • Leave at least one spare T-connector for future expansion.
  • Label both ends of every drop cable.
  • Test the network before permanently securing the wiring.

Engine & Fuel System Integration

This diagram shows how the engine monitoring, fuel monitoring, and NMEA 2000 network work together to provide both reliable analog backup instrumentation and modern digital engine and fuel data. The analog gauges remain the primary backup if the NMEA 2000 network or chartplotter is unavailable, while the Actisense EMU-1 converts engine and fuel tank sender information into NMEA 2000 data for display on the Garmin GPSMAP chartplotter. The Garmin GFS 10 measures actual fuel consumption and sends fuel flow information directly to the NMEA 2000 backbone, allowing the chartplotter to calculate fuel economy, fuel used, fuel remaining, and estimated cruising range.

Installation Notes

  • Install and verify all analog engine gauges first.
  • Connect the engine sensors to the analog gauges as originally designed.
  • Install both fuel tank senders and verify each analog fuel gauge operates correctly.
  • Install the manual fuel selector valve and verify proper operation.
  • Install the Actisense EMU-1 behind the helm where the engine and fuel sender wiring terminates.
  • Connect the following to the EMU-1: Tachometer signal, Oil pressure sender, Coolant temperature sender, Alternator voltage, Port fuel tank sender, and Starboard fuel tank sender.
  • Connect the EMU-1 to the NMEA 2000 backbone using a standard Micro-C drop cable.
  • Configure the EMU-1 and verify all engine and fuel tank data is displayed correctly on the Garmin GPSMAP chartplotter.
  • Install the Garmin GFS 10 module on the NMEA 2000 backbone.
  • Install the GFS 10 inline fuel flow sensor in the fuel hose between the manual fuel selector valve and the engine fuel pump, ensuring it measures fuel flow from either tank.
  • Connect the fuel flow sensor to the GFS 10 module and configure the chartplotter for the engine's fuel tank capacity and fuel flow calibration.
  • Perform a sea trial and verify engine data, fuel levels, fuel flow, fuel economy, and fuel range calculations.

Engine & Fuel System Integration

Recommended Installation Order (Priority)

  • Install and verify analog engine gauges (independent backup instrumentation).
  • Install and verify analog fuel gauges for both fuel tanks.
  • Install the manual fuel selector valve and verify proper fuel delivery from both tanks.
  • Install the NMEA 2000 backbone and network power.
  • Install the Garmin GPSMAP 942xsv and verify network communication.
  • Install the Actisense EMU-1 and connect all engine and fuel tank sender inputs.
  • Configure and test the EMU-1 so engine and fuel tank information appears on the chartplotter.
  • Sea trial the boat to verify all engine monitoring and fuel level functions.
  • Install the Garmin GFS 10 module on the NMEA 2000 network.
  • Install the GFS 10 inline fuel flow sensor between the manual fuel selector valve and the engine fuel pump.
  • Calibrate the fuel flow system and verify fuel consumption, fuel economy, fuel used, and remaining range calculations.
